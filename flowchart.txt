================================================================================
DATA FLOW
================================================================================

1. USER VISITS index.html
   └── Browser loads style.css → imports all CSS modules
   └── Browser loads script.js → imports all JS modules
   └── Modules initialize when DOM ready

2. QUIZ INTERACTION
   └── quiz.js fetches quiz-questions.json
   └── User answers → score tracked in memory
   └── Feedback displayed via updateGameInfo()

3. LOGIN FLOW
   └── User enters name/class in login.html
   └── data-validator.js validates entry data
   └── logbook.js stores to localStorage (student-logbook-entries)
   └── backup-manager.js creates daily backup (student-logbook-backup-YYYY-MM-DD)
   └── backup-compressor.js can create compressed backups (saves 30-40% memory)
   └── backup-compressor.js can create incremental backups (only stores changes)
   └── backup-manager.js maintains archive copy (student-logbook-archive)
   └── logbook-stats.js updates statistics
   └── Login details NOT displayed on page (privacy protection)
   └── Vault can read from localStorage + backups (if authenticated)

4. VAULT ACCESS
   └── User enters password → vault.js hashes with SHA-256
   └── Compares with stored hash in sessionStorage
   └── If match → displays entries from localStorage
   └── User can export (CSV/JSON via data-migrator.js) or search/filter
   └── backup-manager.js provides restore functionality
   └── logbook-stats.js provides analytics and statistics

5. THEME SWITCHING
   └── theme-toggle.js toggles body.light-theme class
   └── CSS variables update via themes.css
   └── Preference saved to localStorage

6. SERVICE WORKER
   └── Registers on first visit
   └── Caches all assets on install
   └── Serves from cache when offline
   └── Updates cache on new version

================================================================================
GAME DIFFICULTY SETTINGS
================================================================================

All games (except impossible.html) have increased difficulty:

1. Circuit Puzzle (puzzle.js)
   - 50 levels (increased from unlimited)
   - Larger grid sizes (up to 8x8, was 7x7)
   - Fewer hints per level (can go to 0 at high levels)
   - More complex path requirements (up to 70% cells cleared)
   - Time limits starting from level 10 (decreasing with level)

2. 3D Memory Matrix (memory3d.js)
   - 100 levels (increased from unlimited)
   - More pairs per level (up to 12 pairs, was 10)
   - Faster flip timeout (minimum 300ms, was 500ms)
   - Time limits starting from level 15 (decreasing with level)

3. Lightning Reflex (reaction.js)
   - 20 waves (increased from 6)
   - Faster target spawn rates (down to 15ms)
   - Shorter target lifetimes (down to 50ms)
   - Increasing accuracy requirements (75% to 90% by wave)
   - More targets per wave (up to 65 targets)
   - Dynamic accuracy checks (60% to 75% minimum)

4. Milestone Match (match-game.js)
   - 50 levels (new level system)
   - Shorter time limits (90s, reduces by 5s per level)
   - Level up every 5 correct answers
   - Lose level on wrong answer
   - More challenging decade matching
   - Stricter scoring

5. Quiz Raga (quiz.js)
   - 50 levels (new level system)
   - Shorter time limits (90s, reduces by 3s per level)
   - Level up every 3 correct answers
   - Lose level on wrong answer
   - Points increase with level (base 8 + level bonus)
   - Penalties increase with level
   - More difficult questions
   - Stricter scoring

6. Quantum Collapse (impossible.js) - UNCHANGED DIFFICULTY
   - Multiple attempts (configurable)
   - Difficulty levels with user choice
   - Progressive difficulty scaling

7. GAME STATISTICS TRACKING
   - game-stats.js tracks all game results
   - Records wins, losses, scores, best times
   - Maintains history for analytics
   - Can be exported for analysis
